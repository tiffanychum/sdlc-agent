## Part 1 — Top 3 Python Async Programming Best Practices

### ✅ Best Practice #1: Always Use `asyncio.TaskGroup` for Concurrent Task Management (Python 3.11+)

**Why it matters:**
`asyncio.TaskGroup` (introduced in Python 3.11, PEP 654) replaces the fragile
`asyncio.gather()` + manual cancellation pattern. It provides **structured concurrency**:
if any child task raises an exception, all sibling tasks are automatically cancelled
and the error is propagated cleanly via `ExceptionGroup`.

**Do this:**
```python
async def main():
    async with asyncio.TaskGroup() as tg:
        task1 = tg.create_task(fetch_data("url_a"))
        task2 = tg.create_task(fetch_data("url_b"))
    # Both tasks are guaranteed to be done here; exceptions are raised together
```

**Avoid this:**
```python
# Old pattern — swallows cancellation, poor error propagation
results = await asyncio.gather(fetch_data("url_a"), fetch_data("url_b"),
                               return_exceptions=True)
```

### ✅ Best Practice #2: Never Block the Event Loop — Offload Blocking I/O to a Thread Pool

**Why it matters:**
Calling any synchronous/blocking function (file I/O, `time.sleep`, CPU-heavy
computation, third-party sync SDKs) inside a coroutine **freezes the entire event
loop**, destroying concurrency. Use `asyncio.to_thread()` (Python 3.9+) or
`loop.run_in_executor()` to push blocking work off the loop.

**Do this:**
```python
import asyncio

async def read_large_file(path: str) -> str:
    # Offloads the blocking open/read to a thread pool worker
    return await asyncio.to_thread(_sync_read, path)

def _sync_read(path: str) -> str:
    with open(path) as f:
        return f.read()
```

**Warning signs in code reviews:**
- `time.sleep(n)` inside a coroutine (use `await asyncio.sleep(n)`)
- Synchronous HTTP calls (e.g., `requests.get(...)`) — replace with `httpx.AsyncClient` or `aiohttp`
- Long CPU loops without `await asyncio.sleep(0)` yield points

### ✅ Best Practice #3: Set Explicit Timeouts on Every Awaitable External Call

**Why it matters:**
Async code is especially prone to **silent hangs**: a network call that never returns
will hold a coroutine open forever, leaking memory and starving the event loop of
capacity. Use `asyncio.timeout()` (Python 3.11+, preferred) or
`asyncio.wait_for()` to enforce hard deadlines.

**Do this:**
```python
import asyncio

async def call_api(url: str) -> dict:
    async with asyncio.timeout(10.0):           # Hard 10-second deadline
        async with httpx.AsyncClient() as client:
            response = await client.get(url)
            return response.json()
```

**Or with `wait_for` (3.9/3.10 compatible):**
```python
result = await asyncio.wait_for(fetch(url), timeout=10.0)
```

**Pitfalls to avoid:**
- No timeout at all (default in `httpx`, `aiohttp`, etc.)
- Catching `asyncio.TimeoutError` and silently retrying without backoff
- Using `httpx.Timeout` on the client but not on individual requests for per-call granularity
